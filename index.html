04

아이콘 리네이밍 원칙

목적

축약 없이, 읽으면 바로 의미가 파악되는 이름으로 개선한다.
AI든 사람이든 이름만 보고 어떤 아이콘인지 추론할 수 있어야 한다는 원칙을 적용한다.

예시  system_completesm  →  system_complete_small
Rule 01
완전한 단어 사용
o, f, sm, lg 같은 약어 대신 outlined, filled, small, large로 풀어써서 맥락 없이도 뜻을 알 수 있게 변경
system_arrowlefto
system_arrow_left_outlined
Rule 02
단어 사이 명확한 구분
붙어있는 단어는 언더스코어로 분리해서 파싱하기 쉽게 변경.
morev2, kebab처럼 시각적 은유나 버전 번호에 의존하는 이름 대신 기능을 직접 설명하게 변경
system_morev2
system_more_horizontal
Rule 03
filled만 표시, outlined 생략
채워진 아이콘만 _filled suffix를 붙이고, 기본(outlined) 아이콘은 suffix 없이 사용. 기본값을 명시하지 않아 이름을 간결하게 유지
system_favorite_star
system_favorite_star_filled
